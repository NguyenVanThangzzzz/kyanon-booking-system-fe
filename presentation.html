Backend — Clean Architecture

4 tầng độc lập · Dependency Rule · SOLID Principles · Constructor DI

1. Domain Layer (Innermost) Entities · Repository Interfaces · Domain Logic Không phụ thuộc bất kỳ framework nào — pure TypeScript
↑
2. Application Layer Services · Use Cases · DTOs Orchestrate domain entities, gọi repositories
↑
3. Infrastructure Layer Prisma Repos · Redis · Logger · JWT Service Implements domain interfaces với framework cụ thể
↑
4. Presentation Layer (Outermost) Controllers · Routes · Zod Validators · Middleware Xử lý HTTP, validate input, format response

↑ Phụ thuộc chỉ hướng vào trong (Dependency Rule)

🔒 Dependency Rule

Inner layers không biết outer layers tồn tại. Domain không import Express hay Prisma.

🔌 Constructor Dependency Injection

Service nhận Repository qua constructor → dễ mock khi test, dễ swap implementation.

🧱 SOLID Principles

SRP: mỗi class một trách nhiệm. OCP: BaseRepository abstract có thể extend. DIP: depend on interfaces, not concretions.

⚠️ Error Hierarchy

AppError → ValidationError (400) · NotFoundError (404) · ConflictError (409) · ForbiddenError (403)

Auth System & Phân quyền

JWT Access + Refresh Token Rotation · Redis blacklist · Role-Based Access Control

Auth Flow

1

Login → Access Token + Refresh Token

Access token JWT ngắn hạn (15 phút). Refresh token lưu PostgreSQL + Redis, xoay vòng mỗi lần dùng.

2

401 → Auto Refresh

Axios interceptor tự động gọi POST /auth/refresh. Các request song song bị 401 được queue lại và replay sau khi refresh thành công.

3

Refresh thất bại → Logout

Server dispatch custom DOM event auth:logout → AuthContext clear localStorage → redirect /login.

4

Logout → Token Revoke

Fire-and-forget revoke request, đồng bộ xóa localStorage.

Role-Based Access (3 Roles)

Tính năng ADMIN STAFF USER
Dashboard tổng quan ✓ ✓ ✗

Đặt phòng (Customer) ✓ ✓ ✓
Rate Limiting: express-rate-limit chặn brute-force. Account tự lock sau N lần đăng nhập sai (lockedUntil field).

Application Services

BookingRefService — sinh mã BK-20260521-AB12CD34
BookingAuthorizationService — kiểm quyền: USER chỉ xem booking của mình

⚡ Redis Distributed Lock

Khi user bắt đầu điền form đặt phòng, server acquire Redis lock cho room:{roomId}.
FE gửi heartbeat mỗi 30s để gia hạn TTL lock.
Nếu user khác cố đặt cùng phòng → trả về 409 Conflict.
Lock tự giải phóng sau khi tạo booking thành công.

📡 SSE — Server-Sent Events

GET /bookings/locks/rooms/:roomId/subscribe
FE mở stream SSE khi phòng bị lock. Server push event room_available ngay khi lock được giải phóng → FE tự động mời user thử lại mà không cần polling.

⏰ Cron Scheduler

pending-booking.scheduler.ts chạy định kỳ:
pending → cancelled khi hết thời hạn thanh toán.
confirmed → no_show khi quá giờ check-in mà khách không đến.

API Endpoints

POST /bookings — Tạo booking (User)
PATCH /bookings/:id/confirm — Xác nhận (Staff)
PATCH /bookings/:id/check-in — Check-in + guest JSONB (Staff)
PATCH /bookings/:id/check-out — Check-out (Staff)
POST /bookings/:id/cancel — Huỷ + tính hoàn tiền
GET /bookings/my — Lịch sử đặt phòng (User)
POST /bookings/locks/heartbeat — Gia hạn Redis lock

Payment Flow — PayOS Integration

QR Code · Webhook · Countdown Timer · Auto status update

① Khách tạo Booking

Chọn phòng, ngày, số khách → tạo booking với status: pending

↓

② Server tạo PayOS Order

POST /api/v1/payments/create → PayOS trả về checkoutUrl + QR Code

↓

③ FE hiển thị QR + Countdown

qrcode.react render mã QR. Timer đếm ngược 15 phút, hết hạn auto cancel.

↓

④ PayOS gửi Webhook

POST /api/v1/payments/webhook → Server xác thực signature → cập nhật PaymentStatus: paid + BookingStatus: confirmed

📊 Booking Status Lifecycle

pending → confirmed → checked_in
completed | cancelled | no_show

🕐 Late Checkout

Khách request trả phòng muộn → Staff/Admin phê duyệt → Kiểm tra conflict với booking kế tiếp → Tính phí phụ nếu có. Status: requested → approved/denied → charged.

⏰ Cron Jobs

node-cron tự động chuyển booking pending → cancelled khi hết thời gian thanh toán và confirmed → no_show khi quá giờ check-in.

Component

↑
🪝

Custom Hook

Quản lý isLoading / error / data. Cancel async bằng cancelled flag.

↑
🌐

Gọi Axios, unwrap API envelope (ApiResponse<T>). Không có state.

🔐

JWT Refresh Token Rotation

PayOS QR Payment

📋

Audit Logging

Immutable log ghi mọi action nhạy cảm (login, tạo user, đổi status, đổi mật khẩu). Lưu cả IP, userAgent và diff changes (old → new).
